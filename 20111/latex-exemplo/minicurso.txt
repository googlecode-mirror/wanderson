\begin{abstract}
This meta-paper describes the style to be used in articles and short papers for
SBC conferences. For papers in English, you should add just an abstract while
for the papers in Portuguese, we also ask for an abstract in Portuguese. In both
cases, abstracts should not have more than 10 lines and must be in the first
page of the paper.
\end{abstract}


% Resumo em Português ----------------------------------------------------------
\begin{resumo}
Este meta-artigo descreve o estilo a ser usado na confecção de artigos e resumos
de artigos para publicação nos anais das conferências organizadas pela SBC. É
solicitada a escrita de resumo e abstract apenas para os artigos escritos em
português. Artigos em inglês deverão apresentar apenas abstract. Nos dois casos,
o autor deve tomar cuidado para que o resumo não ultrapasse 10 linhas cada,
sendo que ambos devem estar na primeira página do artigo.
\end{resumo}

\section{Introdução} % Este é o Nome da Seção

Este documento é um pequeno exemplo de como podemos criar um documento conforme
as normas da Sociedade Brasileira de Computação utilizando \LaTeX{}. Você pode
notar que não estamos formatando as informações em nenhum local, apenas
\emph{marcando} blocos de texto conforme o seu significado.


% Subseção Teoria --------------------------------------------------------------
\subsection{Teoria} % Este é o Nome da Subseção
\label{sec:introducao.teoria} % Esta é a Etiqueta da Seção

Com o \LaTeX{}, não é necessário preocupação com a formatação. Deixamos que o
motor de código aplique os estilos do documento conforme solicitado. No nosso
caso, estamos utilizando um pacote para formatar este documento com o nome
\texttt{sbc-template} que contém informações sobre o modelo. Não é necessário
modificá-lo, basta colocar este arquivo no mesmo diretório do seu documento
atual e trabalhar normalmente. Também são necessários os arquivos
\texttt{caption2.sty} para auxiliar o pacote em alguns estilos específicos e
\texttt{sbc.bst} para formatar as referências bibliográficas.

O LaTeX{} é uma ferramenta antiga. O \TeX{} foi criado no início dos anos 70 por
Donald Knuth. Ele estava insatisfeito com a má qualidade de suas publicações
impressas. Buscando melhorias, inventou o \TeX{}. Leslie Lamport no meio dos
anos 80 queria transformar o \TeX{} em algo mais fácil de ser trabalhado e
desenvolveu um conjunto de macros chamada \LaTeX.

Você não precisa ser um usuário avançado em computação para utilizar o \LaTeX{}.
Basta você ter em mãos poucas ferramentas, como:

\begin{itemize}
    \item Editor de texto simples; e
    \item Tradutor da Linguagem LaTeX.
\end{itemize}

Seguindo a lógica de desenvolvimento, precisamos seguir poucos passos para
concluir o desenvolvimento do documento.

\begin{enumerate}
    \item Idealizar o documento;
    \item Colocar suas principais idéias no arquivo;
    \item Marcar os elementos conforme seus significados; e
    \item Solicitar ao LaTeX{} a formatação.
\end{enumerate}

\section{Referências Bibliográficas} % Este é o Nome da Seção
\label{sec:ref-bibliograficas} % Esta é a etiqueta da Seção


Os documentos técnicos normalmente necessitam de referências bibliográficas.
Basicamente, existe uma ferramenta chamada Bib\TeX{} que também gerencia as
referências do documento. Basicamente você deve incluir um arquivo de nome
\texttt{document.bib}, ou um outro qualquer com a mesma extensão, incluir
algumas informações e após informar o arquivo ao final do documento, você pode
citar as referências dentro do próprio texto.
